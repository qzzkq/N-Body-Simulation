## ✨ Ключевые возможности

* 📦 **Гибкий импорт:** Загрузка начальных условий из HDF5 (`.h5`), `TXT` или случайная генерация систем любого масштаба.
* 🚀 **Четыре алгоритма обсчёта:** От прецизионного симплектического интегратора для Солнечной системы до Barnes–Hut на CUDA для сотен тысяч тел.
* 🕒 **Два режима симуляции:**
* **Real-time:** Интерактивное окно с возможностью изменения масштаба времени.
* **Bake:** Фоновый высокоскоростной просчёт физики напрямую в файл без затрат ресурсов на рендеринг.


* 🎬 **Запись и Replay:** Экспорт результатов для плавного воспроизведения через встроенный плеер.

## 🛠️ Сборка проекта

### Зависимости

* **Обязательные:** C++17 компилятор, OpenGL, GLFW, GLEW, GLM, HDF5 (C++ API), OpenMP.
* **Опционально:** CUDA Toolkit (для GPU-ускорения).

### 🐧 Linux (Makefile)

```bash
cd project

# Сборка (по умолчанию USE_CUDA=1)
make USE_CUDA=0  # CPU-only вариант, если CUDA не установлена
./Simulate

# Сборка остальных утилит
make replay
make datacreator

```

### 🪟 Windows (CMake + MSYS2)

Рекомендуется использовать окружение **MSYS2 MINGW64**. Склонируйте репозиторий в домашнюю директорию (`/home/username/`).

1. Установите компилятор, CMake, Ninja/Make и библиотеки через `pacman`.
2. Выполните сборку:

```bash
cd project
cmake -S . -B build -G "Ninja"
cmake --build build

```

## 🚀 Запуск и CLI-режим

Все три исполняемых файла (`Simulate`, `Replay`, `DataCreator`) поддерживают два режима работы:

1. **Интерактивный:** Запуск без аргументов (программа сама задаст все вопросы в консоли).
2. **CLI-режим:** Активируется при передаче хотя бы одного флага. Интерактивный ввод отключается, используются переданные флаги и дефолтные значения.

### Быстрый старт (Quick Start)

```bash
# Солнечная система через WHFast в интерактивном окне
./Simulate --txt data/solar_nasa.txt --algo 4 --dt 0.00274 --target 100

# Бейк столкновения двух галактик (300k тел, CUDA, без окна визуализации)
./Simulate --realtime 0 --algo 3 --scenario 2 --bodies 300000 --dt 0.01 --target 200 --save_every 5 --output galaxy_collision

# Просмотр сохранённого результата
./Replay data/galaxy_collision.h5

```

### Параметры командной строки `Simulate`

| Флаг | Значения | По умолчанию | Описание |
| --- | --- | --- | --- |
| `--realtime` | `0` / `1` | `1` | `1` — окно с визуализацией; `0` — пакетный режим (Bake, только запись). |
| `--render` | `0` / `1` / `2` | `1` | Только при `--realtime 1`: `0` — кубы, `1` — сферы, `2` — точки (быстрее всего). |
| `--algo` | `1` / `2` / `3` / `4` | `1` | `1` — Brute-force Yoshida4; `2` — Barnes–Hut CPU; `3` — Barnes–Hut CUDA; `4` — WHFast. |
| `--dt` | `float` | `0.000273785` | Фиксированный шаг времени в годах (`0.000273785` $\approx$ 0.1 дня). |
| `--target` | `float` | `10.0` | Время автоостановки в годах (работает только в режиме `--realtime 0`). |
| `--source` | `0` / `1` / `2` | `2` | Источник: `0` — HDF5, `1` — TXT, `2` — случайная генерация. |
| `--h5` | `path` | `""` | Путь к файлу сценария `.h5` (при `--source 0`). |
| `--txt` | `path` | `""` | Путь к файлу конфигурации `TXT` (при `--source 1`). |
| `--scenario` | `0` / `1` / `2` | `0` | Сценарий генерации: `0` — Кольцо, `1` — 8 Кластеров, `2` — Столкновение галактик. |
| `--bodies` | `int` | `100` | Количество генерируемых тел. |
| `--output` | `name` | `frames` | Имя выходного файла траекторий (сохраняется в `data/<name>.h5`). |
| `--save_every` | `int` $\ge 1$ | `10` | Запись в файл каждого N-го шага. `1` — максимальная детализация (огромный файл). |

#### Тонкая настройка точности (Опционально)

| Флаг | По умолчанию | Описание |
| --- | --- | --- |
| `--quality` | `balanced` | Пресет точности: `fast` / `balanced` / `precise` (или `1` / `2` / `3`). Применяется перед оверрайдами ниже. |
| `--bh-theta` | *из пресета* | Параметр открытия ячейки в Barnes–Hut ($\theta$). Больше значение $\rightarrow$ быстрее, но грубее. Обычно `0.5–0.8`. |
| `--softening` | *из пресета* | Plummer-смягчение $\varepsilon$ [AU] в законе силы: $\frac{1}{(r^2+\varepsilon^2)^{3/2}}$. Защищает от деления на ноль при сближении. |
| `--adaptive-eta` | *из пресета* | Коэффициент $\eta$ в уравнении адаптивного подшага $h \le \eta \cdot \sqrt{\dots}$ (для Barnes–Hut CPU). |
| `--substep-cap` | *из пресета* | Максимально допустимое количество подшагов на один кадр интеграции. |

### Утилиты Replay и DataCreator

#### Replay

Предназначен для плавного воспроизведения сохраненных симуляций. Поддерживает таймлайн, перемотку, паузу и повторы. Принимает единственный аргумент — путь к файлу:

```bash
./Replay data/galaxy_collision.h5

```

#### DataCreator

Интерактивный консольный редактор и генератор начальных условий (TXT/HDF5). Запускается строго без аргументов:

```bash
./DataCreator

```

🛑 **Прерывание работы:** Нажатие `Ctrl+C` в любой момент корректно прерывает процесс бейкинга и закрывает `.h5` файл — все вычисленные к этому моменту фреймы сохранятся без повреждения структуры данных.

## 🔬 Физический движок и алгоритмы

### Выбор алгоритма под задачу

* **`--algo 1` (Brute-force Yoshida4):** Оптимально для $N \le 5 \cdot 10^3$. Высочайшая точность, симплектический интегратор 4-го порядка. Дрейф энергии практически нулевой даже на сверхдлинных дистанциях.
* **`--algo 2` (Barnes–Hut CPU):** Оптимально для $N \le 5 \cdot 10^4$ при отсутствии дискретного GPU. Использует схему Velocity-Verlet с адаптивным подшагом.
* **`--algo 3` (Barnes–Hut CUDA):** Разработан для масштабных симуляций $N \ge 10^5$. Единственный практичный выбор для просчёта столкновения галактик (300k+ тел).
* **`--algo 4` (WHFast):** Идеально для Солнечной системы или систем типа «одна тяжелая звезда и легкие спутники». Кеплеровская часть вычисляется аналитически, благодаря чему энергия сохраняется на 1–2 порядка лучше, чем в Verlet.

### Шпаргалка по выбору `--dt`

| Сценарий симуляции | Выбранный `--dt` (годы) | Реальный масштаб шага |
| --- | --- | --- |
| Солнечная система (алгоритм **WHFast**) | `0.00274` | $\approx$ 1 сутки |
| Солнечная система (**Verlet / Yoshida4**) | `0.000274` | $\approx$ 0.1 суток |
| Звездные кластеры / Протопланетные кольца | `0.001 – 0.01` | Подбирается визуально под масштаб |
| Столкновение галактик ($300k+$ тел) | `0.005 – 0.02` | Акцент на общую стабильность, а не микрофизику |

> ⚠️ **Важное замечание:** Симплектичность и сохранение энергии интеграторов гарантируются только при **фиксированном** шаге `dt`. Если кадры в реальном времени рендерятся с плавающей частотой (интервалы между кадрами меняются), внутренний аккумулятор физического движка автоматически разбивает это дельту времени на строго фиксированные подшаги `--dt`.

## 📊 Форматы данных (HDF5)

Вся физика оперирует астрономическими единицами: **расстояние** — А.Е., **скорость** — А.Е./год, **масса** — $M_\odot$ (масса Солнца), **радиус** — км.

### Входные сценарии (`.h5`)

Должны содержать структурированный датасет `Particles` (Compound Type) со следующими полями:

* `position`: `double[3]`
* `velocity`: `double[3]`
* `mass`: `double`
* `radius`: `double`

### Выходные кадры симуляции (`frames.h5`)

Генерируются автоматически для последующего воспроизведения:

* **Глобальные атрибуты:** `num_bodies`, `dt` (время между кадрами), `num_frames`.
* **`initial_data`:** Датасет со стартовым физическим состоянием всех объектов.
* **`tracks`:** Расширяемый трехмерный массив, хранящий точные координаты тел для каждого сохраненного кадра.
