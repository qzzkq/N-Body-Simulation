# 🌌 N-Body Simulation (CPU / Barnes–Hut / CUDA)

Интерактивный 3D-симулятор гравитационного N-body взаимодействия на C++17 с высокопроизводительной визуализацией через `OpenGL`

**Доска задач** проекта: https://github.com/users/qzzkq/projects/1/views/1

---

## ✨ Ключевые возможности

* 📦 **Гибкий импорт:** Загрузка начальных условий из HDF5 (`.h5`), `TXT` или случайная генерация систем любого масштаба.
* 🚀 **Три алгоритма обсчёта:**
* **Brute Force (CPU):** Точный расчёт $O(N^2)$ с интегратором RK4.
* **Barnes–Hut (CPU):** Оптимизация $O(N \log N)$ на основе октодерева.
* **Barnes–Hut (CUDA):** Максимальное ускорение вычислений на GPU.


* 🕒 **Два режима симуляции:**
* **Real-time:** Интерактивный просмотр с возможностью ускорения/замедления времени.
* **Bake:** Быстрый фоновый просчёт физики до лимита времени с записью траекторий.


* 🎬 **Запись и Replay:** Экспорт результатов в `data/frames.h5` для последующего плавного воспроизведения через плеер `replay`.

## 🛠️ Сборка проекта

### Зависимости

* **Обязательные:** C++17 компилятор, OpenGL, GLFW, GLEW, GLM, HDF5 (C++ API).
* **Опционально:** CUDA Toolkit (для GPU ускорения).

### 🪟 Windows (CMake + MSYS2)

Рекомендуется использовать окружение **MSYS2 MINGW64**. Склонируйте репозиторий в домашнюю директорию (`/home/username/`).

1. Установите компилятор, CMake, Ninja/Make и необходимые библиотеки через `pacman`.
2. Выполните сборку:

```bash
cd project
cmake -S . -B build
cmake --build build

```

## 🚀 Запуск и сценарии

После старта `./Simulate` предложит настроить симуляцию в интерактивном консольном меню:

1. **Отрисовка:** Шары `[1]` / Кубы `[0]` / Точки `[2]`.
2. **Алгоритм:** Brute Force `[1]` / Barnes–Hut CPU `[2]` / Barnes–Hut CUDA `[3]`.
3. **Сценарий:** Загрузка готового `.h5` из `data/` `[0]` или генерация $N$ тел `[1]`.
4. **Режим:** Real-time `[1]` / Режим Bake `[0]` (потребуется ввести конечное время).

> 💾 Результаты симуляции автоматически сохраняются и перезаписывают файл `data/frames.h5`.

## 📊 Форматы данных (HDF5)

В коде используются астрономические единицы: **расстояние** — А.Е., **скорость** — А.Е./год, **масса** — $M_\odot$ (масса Солнца), **радиус** — км.

### Входные сценарии (`.h5`)

Должен содержать датасет `Particles` со следующей структурой структуры (Compound Type):

* `position`: `double[3]`
* `velocity`: `double[3]`
* `mass`: `double`
* `radius`: `double`

*Готовые пресеты (например, `solar_system.h5`) находятся в папке `data/`.*

### Выходные кадры (`frames.h5`)

Создаётся автоматически для плеера `replay`:

* **Атрибуты:** `num_bodies`, `dt` (интервал кадра), `num_frames`.
* **`initial_data`:** Стартовое состояние системы.
* **`tracks`:** Расширяемый массив координат всех тел по кадрам.

Запуск воспроизведения:

```bash
./replay data/frames.h5

```
